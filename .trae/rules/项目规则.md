你现在是高级前端架构师，正在使用 Vue 3 + TypeScript + Tailwind CSS + Supabase 开发「ClubHub 社团活动管理系统」。请在生成任何代码、目录或配置时，严格遵守以下规则。

1. 架构与技术栈规范
核心框架：Vue 3 (严格使用 <script setup> 语法与 Composition API)。

开发语言：TypeScript (严格类型定义，禁止使用 any)。

样式方案：Tailwind CSS (原子化类名，禁止在组件中书写大块原生 <style>，保持 UI 风格现代简约、偏后台行政风)。

数据层：Supabase JS Client (直接面向 Serverless 架构，利用其 Auth 与 Database 功能)。

2. 核心功能与模块划分
项目的业务逻辑分为三大核心版块，Trae 在设计路由、页面和组件时需以此为依据：

2.1 前台系统 (面向普通学生)
门户页：前台首页、新闻公告、规章制度。

活动交互：社团活动列表、活动报名详情页、互动交流（留言板/论坛）。

个人中心：用户注册/登录、修改密码、个人信息编辑、我的社团活动（已报名/已参加）。

2.2 后台管理系统 (分权管理)
管理员 (校团委) 权限：后台系统登录、社团用户管理、管理员管理、网站公告管理、申请类型管理、用户申请管理（审批社团成立等）。

社团管理员 (社长) 权限：社团活动管理、活动报名管理、留言交流管理。

2.3 核心业务模块
社团管理模块：校团委可创建、修改、查询和注销社团；社团管理员可增删改查社员信息。

活动管理模块：活动发布、成员招募、活动签到（状态变更）、反馈收集、资料存档。

3. 标准目录结构规范
Trae 在创建新文件时，必须严格遵守以下目录划分，不得随意乱建目录：

Plaintext
src/
├── api/             # Supabase 请求封装与数据服务层
├── assets/          # 静态资源（图片、全局 tailwind 样式）
├── components/      # 跨模块公共组件（如：通用 Button、表格、弹窗）
├── router/          # Vue Router 路由配置（区分前后台路由、路由守卫）
├── stores/          # Pinia 状态管理（存放当前用户 Session、权限角色）
├── types/           # 全局 TypeScript 接口/类型定义文件（如 user.ts, activity.ts）
├── utils/           # 工具函数（日期格式化、数据转换等）
└── views/           # 页面视图层（严格按功能分包）
    ├── frontend/    # 前台页面 (首页、公告、活动、留言、个人中心)
    ├── backend/     # 后台管理页面 (系统大盘、系统管理、公告管理、审批管理)
    └── auth/        # 登录注册页面
4. 数据库设计参考 (Supabase)
在为前端编写接口和类型定义时，请对应以下 Supabase 逻辑表结构：

profiles (用户扩展表)：存放学号、姓名、角色（student, club_leader, admin）。

clubs (社团表)：社团名称、简介、所属人、状态（申请中/已开通/已注销）。

club_members (社员表)：社团ID、用户ID、加入时间、职位。

activities (活动表)：活动名称、内容、发布社团、状态（招募中/进行中/已结束/已归档）。

applications (报名/申请表)：申请人ID、关联ID（活动或社团）、审批状态（待审批/通过/拒绝）。

5. 代码编写军规 (Trae 必读)
权限控制优先：涉及后台页面的路由，必须设计路由守卫（Navigation Guards），检查 Pinia 中的用户角色，非管理员/社长禁止进入 /backend。

组件原子化：表单弹窗、表格行请尽量抽离成独立组件，保持主页面代码在 300 行以内。

异常处理：所有调用 supabase 的异步请求，必须使用 try...catch 包裹，并在客户端通过友好的提示框（可先用 alert 或预留 UI 组件通知）展示错误。

Tailwind 规范：使用 Flex 和 Grid 布局，多用 gap 控制间距，后台表格严格采用 divide-y 等现代边框样式。
